# Deep audit — July 2026 (v0.17 → v0.18)

Four parallel adversarial audits (runtime correctness, parser robustness,
intelligence-layer logic/honesty, test coverage) swept the whole codebase.
This records what was found, what was fixed in v0.18.0, and what remains
open. Three headline bugs were reproduced live before fixing.

## Fixed in v0.18.0 (with regression tests in tests/test_audit_fixes.py)

**Map-wrong / silent-corruption:**
- **NL CRS false positive** — a bare `3857`/`4326` anywhere in a description
  (e.g. inside `select where "POP > 3857"`) hijacked the CRS. Now only phrase
  forms (`web mercator`, `wgs84`) and `epsg:NNNN` set the CRS.
- **Intent leading clip/erase** — "clip to boundary" on uploaded data clipped
  the boundary *by itself*; the data layer was never clipped. The operation
  subject is now the originally-uploaded data layer (captured before
  instruction-named files are added).
- **Intent silent redirect** — an operation naming a layer that wasn't
  uploaded (`buffer rivers…`) silently ran on the default layer. Now the
  named-but-absent subject is kept unresolved so validation surfaces it
  (stopword-guarded so "buffer by 500 m" isn't mistaken for a layer).
- **Op-output identifiers** — an operation output containing a space/hyphen/
  leading digit made the whole generated script a SyntaxError (generation
  aborted). Outputs are now normalised to valid identifiers, references and
  layer names updated to match.

**Overclaim (asserting more than the rules justify):**
- **Substring driver matching** — "rain" ⊂ "terrain", "road" ⊂ "railroad",
  "city" ⊂ "capacity" etc. made the systems layer and discover claim drivers/
  themes the data didn't have. Now leading-boundary matching (`_kw_hit`).
- **Diverging-ramp detection** — a fixed mid-index + `>200` test wrongly
  called `spectral`, `red_yellow_green`, `sensitivity` (6-colour / pale
  centre) "sequential", firing false warnings on correct choices. Now
  luminance-based: diverging iff the lightest colour is interior.
- **ESZ archetype** — carried neither the projected-CRS requirement nor the
  Supreme Court 26-Apr-2023 caveat the study mandates, and overrode an
  explicit user ramp. Now: the note carries the legal caveat + CRS rule, a
  geographic-CRS warning fires, and the ring renderer respects `using X`.
- **classify_pair convergence** — two actors *converging* were labelled
  "success to the successful" (the opposite). Now S2S requires divergence
  *from parity*, not mere endpoint movement.
- **discover false themes** — same substring issue (rain⊂drainage, emission⊂
  transmission); fixed with leading-boundary matching. Also: the change-map
  suggestion now fires for the 2-epoch case (its canonical input), and its
  `why` states the co-registration requirement instead of asserting it.

**Runtime / arcpy (correct-by-docs; still need a Pro shakedown):**
- **KML loader** guessed feature-class names that don't exist; every KML/KMZ
  silently loaded nothing. Now walks the output gdb (`arcpy.da.Walk`).
- **.jpg export** wrote PNG bytes to a .jpg file; now uses `exportToJPEG`.
- **NetCDF series means** — `GetRasterProperties MEAN` on a raw .nc / a
  stats-less raster raised; now builds statistics first and degrades
  gracefully with a clear NetCDF hint.

## Open — documented, not yet fixed (honest limitations / lower severity)

- **No mock-arcpy harness** — the runtime is executed for real only inside
  Pro; CI checks signatures by string presence, not by running the generated
  script. The single highest-value next test-infra investment.
- **set_extent SR mismatch** (runtime) — a WGS84 bbox applied to a projected
  map frame may not reproject reliably; extent-zoom can land wrong. Needs a
  Pro run to confirm; a projection of the bbox to the map SR is the fix.
- **apply_graduated with std_dev/defined_interval** — those methods derive
  their own class count, so setting `breakCount` may raise → default
  symbology. Needs Pro validation.
- **_surround style tokens** — `listStyleItems("ArcGIS 2D", "North_Arrow")`
  may need the upper-snake style_class token; surrounds may fall back to
  default styling. Pro-run dependent.
- **`.prj` EPSG extraction** — Esri WKT often omits `AUTHORITY`, so the common
  ArcGIS UTM shapefile yields no EPSG and the bbox heuristic guesses (flagged
  in a note, not silent). A `UNIT[...AUTHORITY["EPSG","9001"]]`-last WKT can
  return the unit code. A real WKT→EPSG resolver (or pyproj, optional) is the
  proper fix; deliberately avoided to keep zero-dependency.
- **CSV coordinate sniffing** — comma-delimited only; `POINT_X`/`E`/`N` not
  recognised; bare `x`/`y` can false-positive. Documented in GAPS.
- **GeoPackage identifier interpolation** — table name goes unescaped into a
  `PRAGMA` f-string; confined to breaking its own read (caught), but should be
  quoted. Low security surface.
- **dynamics thresholds** — 0.9 R², 0.55 fraction-of-K, 0.92 overshoot etc.
  are uncited and fragile to a single noisy tick; the identifiability caveat
  is on every positive `classify` result. Behaviour classification, not proof.
- **Parser error contract** — CIM/GeoJSON `json.load` and a non-UTF-8 `.json`
  can raise uncaught (vs the clean `ValueError` the KML/GPX/CSV paths raise);
  the server wraps `(ValueError, FileNotFoundError)` so most cases are clean
  400s, but `struct.error` on hostile binary uploads could 500.
- **ZIP extraction collisions** — two datasets sharing their last two path
  components clobber on disk; nested zips aren't recursed.
- **examples/ not in the sdist** — a pip-installed user can't run the
  file-based examples (built-in NL examples still work).

These are the register; each is a scoped next task, not a hidden defect.
